════════════════════════════════════════
  PART 13: DEPLOYING FROM ANOTHER LAPTOP
════════════════════════════════════════

If you switch to a new PC/laptop, here's what you need to do:

--- STEP 1: Install Prerequisites ---
  - Git:     https://git-scm.com
  - Node.js: https://nodejs.org
  (PHP + Composer only needed if testing Laravel locally)

--- STEP 2: Clone the Repo ---
  Open PowerShell:
  git clone https://github.com/YOUR_USERNAME/dental-clinic-system.git
  cd dental-clinic-system

--- STEP 3: Set Up SSH Key (needed for frontend SCP upload) ---

  OPTION A — Copy key from original PC (easiest):
  1. On your OLD PC, run:
       Get-Content "$HOME\.ssh\hostinger_deploy"      ← private key
       Get-Content "$HOME\.ssh\hostinger_deploy.pub"  ← public key
  2. On the NEW laptop:
       New-Item -ItemType Directory -Force "$HOME\.ssh"
       notepad "$HOME\.ssh\hostinger_deploy"      ← paste private key, save
       notepad "$HOME\.ssh\hostinger_deploy.pub"  ← paste public key, save
       icacls "$HOME\.ssh\hostinger_deploy" /inheritance:r /grant:r "$env:USERNAME:R"

  OPTION B — Generate a NEW key on new laptop (more secure):
  1. Run:
       ssh-keygen -t rsa -b 4096 -C "deploy-laptop2" -f "$HOME\.ssh\hostinger_deploy"
  2. Add the new PUBLIC key to Hostinger:
       hPanel → Advanced → SSH Access → Add SSH Key
       paste contents of: "$HOME\.ssh\hostinger_deploy.pub"
  3. Update GitHub Secret SSH_PRIVATE_KEY with the new private key:
       GitHub repo → Settings → Secrets and variables → Actions
       → Edit SSH_PRIVATE_KEY → paste contents of "$HOME\.ssh\hostinger_deploy"

--- STEP 4: Deploy ---

  BACKEND (automatic — no extra setup):
    git add . && git commit -m "Update" && git push
    ✅ GitHub Actions handles the SSH to Hostinger automatically

  FRONTEND (needs SSH key from Step 3):
    cd frontend-react
    npm install
    npm run build
    scp -i "$HOME\.ssh\hostinger_deploy" -P 65002 -r dist/* u940130461@37.44.245.132:~/domains/mcdzn3.com/public_html/

  ML SERVICE (automatic — no extra setup):
    git push → Render auto-deploys ✅

--- SUMMARY ---
  Backend  → just git push, GitHub Actions does everything
  Frontend → needs Node.js + SSH key (Option A or B above)
  ML       → just git push, Render does everything
  ⚠️  The SSH private key ($HOME\.ssh\hostinger_deploy) is the only
      thing that does NOT transfer automatically to a new machine.
