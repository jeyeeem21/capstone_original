6. BUSINESS PROCESS FLOW (Procurement → Sale)
================================================================================

    ┌─────────────────────────────────────────────────────────────────┐
    │  STAGE 1: PROCUREMENT (Buy Raw Rice from Supplier)            │
    │                                                                │
    │  Input: Supplier + Variety + Quantity(kg) + Sacks + Price/kg   │
    │  Formula: total_cost = quantity_kg × price_per_kg              │
    │  Example: 5,000 kg × ₱10/kg = ₱50,000                        │
    │  Status: Pending                                               │
    │  Optional: Assign to Procurement Batch                         │
    └──────────────────────────┬──────────────────────────────────────┘
                               │
                               ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │  STAGE 2: DRYING (Sun-dry or Machine-dry the Raw Rice)        │
    │                                                                │
    │  Create from: Individual procurement OR entire batch           │
    │  Input: sacks, price_per_sack_per_day                          │
    │  Daily: Admin clicks "Increment Day" → days += 1              │
    │  Formula: drying_cost = sacks × price_per_sack × days          │
    │  Example: 40 sacks × ₱50/sack × 3 days = ₱6,000              │
    │  Status: Drying → Dried (click "Mark as Dried")               │
    │  Can Postpone if needed                                        │
    │  Delete only if quantity_out = 0                               │
    └──────────────────────────┬──────────────────────────────────────┘
                               │
                               ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │  STAGE 3: PROCESSING / MILLING (Hull & Mill the Dried Rice)   │
    │                                                                │
    │  Select: one or more Dried drying processes as input sources   │
    │  Input: total input_kg to mill                                 │
    │  System: proportionally splits across selected drying sources  │
    │  Status: Pending → Processing → Completed                      │
    │  On Complete: enter output_kg, operator_name                   │
    │  Auto-calc: husk_kg = input - output                           │
    │             yield% = (output / input) × 100                    │
    │  Example: 1,000 kg in → 600 kg out = 60% yield               │
    │  Revert to Processing: only if stock_out = 0                  │
    │  Delete: only if stock_out = 0                                │
    └──────────────────────────┬──────────────────────────────────────┘
                               │
                               ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │  STAGE 4: STOCK DISTRIBUTION (Bag into Product Units)         │
    │                                                                │
    │  Select: completed processing(s) matching product's variety    │
    │  Input: kg to take from each processing source                │
    │  System calculates:                                            │
    │    total_units = floor(total_kg / product_weight_per_unit)     │
    │                                                                │
    │  COST TRACING (Automated):                                     │
    │    fraction = kg_taken / total_drying_kg                       │
    │    procurement_cost_share = fraction × procurement_total       │
    │    drying_cost_share = fraction × drying_total_price           │
    │    total_production_cost = procurement + drying costs           │
    │    cost_per_unit = total_cost / units                          │
    │    profit_per_unit = selling_price - cost_per_unit             │
    │    profit_margin% = (profit / price) × 100                    │
    │                                                                │
    │  Visual Indicators:                                             │
    │    🟢 Green  = 20%+ margin (healthy)                           │
    │    🟡 Amber  = 0-20% margin (warning)                          │
    │    🔴 Red    = Negative margin (selling below cost!)            │
    │                                                                │
    │  Stock Logs created with full cost breakdown                   │
    └──────────────────────────┬──────────────────────────────────────┘
                               │
                               ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │  STAGE 5: SALES (Sell via POS or Online Orders)               │
    │                                                                │
    │  A) POS (Walk-In):                                             │
    │     - Add products to cart                                     │
    │     - Select/create customer                                   │
    │     - Payment: Cash or GCash                                   │
    │     - Transaction ID: TXN-YYYYMMDD-NNN                        │
    │     - Status: completed (stock deducted immediately)           │
    │                                                                │
    │  B) ORDERS (Online/Phone):                                     │
    │     - Create order for customer                                │
    │     - Payment: Cash/GCash/COD/Pay Later                        │
    │     - Order ID: ORD-YYYYMMDD-NNN                              │
    │     - Status: pending (stock deducted at processing)           │
    │                                                                │
    │  ORDER STATUS FLOW:                                            │
    │  Pending → Processing → Shipped → Delivered                    │
    │     │         │            │          │                         │
    │     └─────────┴────────────┴──────────┴─→ Voided (restock)     │
    │                                                                │
    │  RETURN FLOW:                                                  │
    │  Delivered → Return Requested → Picking Up → Returned(restock) │
    │                    │                                            │
    │                    └──→ Rejected (back to Delivered)            │
    └────────────────────────────────────────────────────────────────┘
