1. SYSTEM OVERVIEW & ROLE DIFFERENCES
================================================================================

The KJP Ricemill system has 5 user roles:
  - Super Admin    → Full system control (all features)
  - Admin          → Operational management (limited settings, no archive/delete)
  - Staff          → POS access + staff dashboard
  - Driver         → Delivery management (to be documented later)
  - Client         → Online shop, ordering, cart (to be documented later)

SUPER ADMIN vs ADMIN — Key Differences:

│ Feature                              │ Super Admin  │ Admin        │

│ Procurement — Add/Edit               │ ✓            │ ✓            │

3. DASHBOARD
================================================================================

URL:        /superadmin/dashboard  OR  /admin/dashboard
Access:     Both Admin and Super Admin
API:        None connected yet (placeholder data)

Description:
  The main overview page showing key business performance metrics and charts.
  Provides a quick snapshot of the ricemill's operations at a glance.

Functions & Features:
  • Stats Cards (4 cards displayed):
      - Total Input (kg palay)
      - Total Output (kg rice)
      - Average Yield (%)
      - Total Records (entries)

  • Line Chart — Input vs Output Trends:
      - Period toggle: Daily / Monthly / Yearly
      - Summary stats: Total Output, Avg per Day, Growth %

  • Donut Chart — Processing Breakdown:
      - Shows output vs waste distribution

  • Recent Sales Table:
      - Columns: Invoice, Customer, Date, Amount, Status
      - Searchable and filterable by status

  • Skeleton loaders during data loading

4. PROCUREMENT
================================================================================

URL:        /superadmin/procurement  OR  /admin/procurement
Access:     Both Admin and Super Admin
            Archive/Delete button is Super Admin ONLY
API:        GET /procurements
            GET /suppliers
            GET /varieties
            GET /procurement-batches
            POST /procurements
            PUT /procurements/:id
            DELETE /procurements/:id

Description:
  Manages palay (unhusked rice) procurement records from suppliers. Supports
  batch grouping for organized procurement and sending to drying process.
  One of the largest pages in the system (~2,270 lines).

Functions & Features:
  • CRUD Operations:
      - Add new procurement record
      - View procurement details (double-click row)
      - Edit existing procurement
      - Delete/archive procurement (Super Admin only)

  • Add Procurement Form Fields:
      - Supplier (combobox with search + "add new supplier" inline option)
      - Quantity in kg
      - Number of sacks
      - Price per kg
      - Description/notes
      - Status
      - Batch assignment (optional)

  • Batch Management:
      - Toggle between "Records" and "Batches" table views
      - Create new procurement batches with notes
      - Filter records by batch
      - View batch details in modal
      - Batch summary showing total sacks, kg, cost

  • Send to Drying:
      - Send individual procurement to drying process
      - Send entire batch to drying process
      - Specify sacks and price per kg for drying
      - Batch preview before sending

  • New Supplier Inline Creation:
      - While adding procurement, create a new supplier on-the-fly

  • Remarks Edit:
      - For already-dried procurements, only remarks/description can be edited

  • Charts:
      - Line chart with Daily/Monthly/Yearly period toggle
      - Donut chart for procurement breakdown

  • Stats Cards showing procurement summary metrics
  • Data table with search, sort, and pagination
  • Skeleton loaders during initial load

6. PROCESSING (MILLING)
================================================================================

URL:        /superadmin/processing  OR  /admin/processing
Access:     Both Admin and Super Admin
API:        GET /processings/active
            GET /processings/completed
            GET /drying-processes
            POST /processings
            PUT /processings/:id
            DELETE /processings/:id

Description:
  Manages rice milling/processing operations — converting dried palay into
  finished rice products. Supports a workflow from pending → active → completed.

Functions & Features:
  • CRUD Operations:
      - Add new processing record
      - View processing details (modal)
      - Edit processing record
      - Delete/archive processing

  • Processing Workflow:
      - Pending → Start Processing (begins milling)
      - Active → Complete Processing (enter output quantities)
      - Completed → Return to Processing (revert if needed)

  • Multi-select Drying Sources:
      - Select multiple dried items as inputs for one processing run

  • Complete Processing Form:
      - Output in kg
      - Husk in kg
      - Yield percentage (auto-calculated)

  • Two Separate Tables:
      - Active Processings table
      - Completed Processings table

  • Context-aware Action Buttons:
      - Pending: Edit, Archive
      - Active: Start, Complete
      - Completed: Return to Processing

  • Charts:
      - Line chart with Daily/Monthly/Yearly period toggle
      - Donut chart for processing breakdown

  • Stats Cards: Total input, output, yield %, records count
  • Skeleton loaders

10. POINT OF SALE (POS)
================================================================================

URL:        /superadmin/pos  OR  /admin/pos  OR  /staff/pos
            Also accessible via /pos (auto-redirects based on role)
Access:     Super Admin, Admin, AND Staff (shared page)
API:        GET /products
            GET /sales
            GET /varieties
            POST /sales/order
            POST /sales/:id/void

Description:
  Real-time point-of-sale interface for processing walk-in and order
  transactions. The primary transaction-creation interface. Shared across
  all internal roles.

Functions & Features:
  • Product Grid:
      - Searchable product cards
      - Filterable by rice variety
      - Displays: Product name, price, stock level, variety badge
      - Click a product card to add to cart
      - Shows all active products including 0-stock items

  • Cart Management:
      - Add products to cart (click product card)
      - Update quantity with +/- buttons
      - Remove individual items
      - Running total and item count displayed
      - Mobile-responsive collapsible cart panel

  • Payment Processing:
      - Cash: Enter cash tendered amount, auto-calculate change
      - GCash: Enter reference number
      - COD: Cash on Delivery option

  • Payment Modal:
      - Validates payment amounts before processing
      - Shows order summary (items, quantities, total)
      - Sends complete order payload to API
      - Validates stock availability before submission

  • Sale Complete Modal (Receipt):
      - Receipt-style summary after successful sale
      - Shows: Transaction ID, items, quantities, total, payment details
      - Displays change amount for cash payments

  • Void Transaction:
      - Search and select completed transactions from today
      - Enter void reason
      - Processes refund and restores stock levels

  • Today's Stats Cards (4):
      - Today's Sales (total revenue)
      - Transactions (order count)
      - Items Sold (unit count)
      - Avg Transaction (average order value)

  • Recent Transactions List:
      - Shows today's transactions
      - Columns: ID, Time, Total, Items, Payment Method, Status

  • Real-time Stock Refresh:
      - Stock levels refresh after each transaction

11. ORDERS
================================================================================

URL:        /superadmin/orders  OR  /admin/orders
Access:     Both Admin and Super Admin
API:        GET /sales
            PUT /sales/:id/status
            POST /sales/:id/return

Description:
  Order management pipeline — tracks in-progress orders from pending through
  delivery, with cancel and return workflows.

Functions & Features:
  • Order Status Pipeline:
      Pending → Processing → Shipped → Delivered
      (One-click status advancement buttons)

  • Status Tab Filters:
      All, Pending, Processing, Shipped, Delivered,
      Return Requested, Returned, Cancelled
      (Color-coded tab buttons with counts)

  • Process Order:
      - Move order from Pending → Processing

  • Ship Order:
      - Move order from Processing → Shipped

  • Deliver Order:
      - Move order from Shipped → Delivered

  • Cancel Order:
      - Confirm modal to cancel an order
      - Stock is automatically restored

  • Return Order (Full Flow):
      - Request Return: Modal with return_reason, return_notes,
        return_proof (multi-image upload)
      - Accept Return: Assign pickup driver, plate number,
        estimated pickup date
      - Reject Return: Reverts order status to delivered
      - Mark as Returned: Pickup complete, stock restored

  • Restock Items:
      - Per-item quantity selection for restocking after void/return
      - Each item tracks `restocked` status to prevent double-restock

  • Record Payment:
      - Mark unpaid orders as paid (cash or GCash + proof upload)
      - Payment proof stored as JSON array of image paths

  • View Order Details:
      - Modal showing all order items, totals, customer info

  • Stats Cards:
      - Total Revenue (from delivered orders)
      - Pending Orders count
      - Delivered Orders count
      - Total Items

  • Charts:
      - Line chart: Revenue trends over time
      - Donut charts: Status breakdown, Payment method breakdown

  • Data Table:
      Columns: Order ID, Customer, Items, Total, Payment, Date, Status, Actions
      Action buttons are context-aware based on current order status

  • Skeleton loaders

20. ARCHIVES
================================================================================

URL:        Embedded within Settings → Archives section
Access:     Super Admin ONLY
API:        GET /archives
            GET /archives/statistics
            POST /archives/:module/:id/restore
            DELETE /archives/:module/:id

Description:
  Manages soft-deleted/archived records across all modules. Records that
  are archived (deleted) from any page appear here for review, restoration,
  or permanent deletion.

Supported Modules:
  Products, Varieties, Suppliers, Customers, Procurements,
  Drying Processes, Processings, Drivers, Deliveries

Functions & Features:
  • Restore Record:
      - Confirm modal to restore an archived record back to active status
      - Automatically restores status (Active for varieties/suppliers/customers,
        Pending for procurements)
      - Invalidates relevant caches after restoration

  • Permanent Delete:
      - Confirm modal to permanently remove a record from the database
      - WARNING: This action CANNOT be undone

  • "Archived By" Tracking:
      - Shows who archived each record (pulled from audit trail)
      - Displays user name with icon, or dash if unknown

  • Module Filter:
      - Dropdown filter to view archives by module type
      - Options: All Modules, Product, Variety, Supplier, Customer,
        Procurement, Drying Process, Processing, Driver, Delivery

  • Module Icons & Colors:
      - Each module has a distinct icon and color badge for identification

  • Stats Cards:
      - Total Archived (record count)
      - Modules Affected (how many module types have archives)
      - Most Recent (most recently archived record)

  • Data Table:
      Columns: Record (name + ID), Module, Archived Date, Archived By, Actions
      Searchable, sortable, with module filter dropdown and date range filter
      Uses composite keys (module:id) to avoid React key collisions

  • Skeleton loaders
