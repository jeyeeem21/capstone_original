0. COST RECORDS PAGE — WHAT IT ALREADY SHOWS (INSIGHTS)
================================================================================

Location: Admin Panel → Inventory → "Costs" tab (4th tab)
Route:    /admin/inventory?tab=costs

This tab already EXISTS in the system and is the closest thing to a
cost/profit report. Understanding what it shows is critical before building
the full Reports page.

── WHAT THE COSTS TAB SHOWS ─────────────────────────────────────────────────

  Each row in the table = one STOCK DISTRIBUTION EVENT
  (i.e., when completed processing output was added to a product's inventory)

  COLUMNS:
  ┌─────────────────┬──────────────────────────────────────────────────────┐
  │ Column          │ What It Means                                        │
  ├─────────────────┼──────────────────────────────────────────────────────┤
  │ Product         │ Product name + variety color dot                     │
  │ Units           │ Bags/units added + total kg + kg per sack            │
  │ Procurement     │ Raw palay cost share for this batch                  │
  │ Drying          │ Drying cost share for this batch                     │
  │ Total Cost      │ Procurement + Drying (COGS for this distribution)    │
  │ Cost / Unit     │ Total Cost ÷ Units added = break-even per bag        │
  │ Sell Price      │ Current selling price per bag (from Products table)  │
  │ Profit / Unit   │ Sell Price − Cost/Unit (green = profit, red = loss)  │
  │ Margin          │ (Profit/Unit ÷ Sell Price) × 100%                    │
  │                 │   Green ≥ 20% | Amber ≥ 0% | Red < 0%               │
  │ Date            │ When the stock was distributed into inventory        │
  └─────────────────┴──────────────────────────────────────────────────────┘

  STATS CARDS AT TOP:
  ┌──────────────────────────┬──────────────────────────────────────────────┐
  │ Card                     │ Formula                                      │
  ├──────────────────────────┼──────────────────────────────────────────────┤
  │ Total Production Cost    │ SUM(total_cost) of all distributions         │
  │ Avg. Cost / Unit         │ Total Cost ÷ Total Units produced            │
  │ Avg. Profit / Unit       │ Average of all (Sell Price − Cost/Unit)      │
  │ Avg. Profit Margin       │ Average margin % + total profit amount       │
  └──────────────────────────┴──────────────────────────────────────────────┘

  DOUBLE-CLICK a row → opens detail modal showing the exact processing
  records that produced the stock, with drying source breakdown.

── DATA SOURCE ───────────────────────────────────────────────────────────────

  All data comes from the `stock_logs` table, filtered to:
    - type = 'in'  (stock additions only)
    - total_cost IS NOT NULL  (only distributions with cost data)

  Fields used from stock_logs:
    procurement_cost, drying_cost, total_cost, cost_per_unit,
    selling_price, profit_per_unit, profit_margin, kg_amount,
    source_type, source_processing_ids

── KEY INSIGHT: SELL PRICE IS LIVE, NOT HISTORICAL ──────────────────────────

  !! IMPORTANT — The "Sell Price" and therefore "Profit/Unit" and "Margin"
     are computed using the CURRENT product price from the Products table.

  This means:
    - If a product's selling price was ₱1,200 when the stock was created,
      but you changed it to ₱1,500 later, the Costs tab will show ₱1,500
      as the sell price, recalculating all profit figures dynamically.
    - This is POTENTIAL profit per unit (at current price), not ACTUAL
      realized profit from historical sales.

  For ACTUAL realized profit (what was truly earned), use:
    - sale_items.unit_price  (the price at time of sale)
    - combined with stock_logs.cost_per_unit (cost at time of production)

── WHAT THE COSTS TAB IS GOOD FOR ───────────────────────────────────────────

  ✓ Knowing the break-even price per bag for each production batch
  ✓ Checking if current selling price covers production costs
  ✓ Identifying which product batches had low or negative margins
  ✓ Tracing full cost chain per stock distribution event
  ✓ Comparing procurement + drying cost split per product

── WHAT THE COSTS TAB IS NOT GOOD FOR ───────────────────────────────────────

  ✗ Knowing total profit for a specific time period (monthly/yearly)
  ✗ Comparing revenue earned vs costs incurred for a period
  ✗ Showing actual sale amounts vs actual production costs
  ✗ Filtering by date range for a specific period report
  ✗ Printing a formatted report for accounting/management

── WHAT NEEDS TO BE ADDED FOR PROPER PROFIT REPORTS ─────────────────────────

  1. The Costs tab should get a DATE FILTER (from/to) so cost records for a
     specific period can be isolated

  2. A "Print Cost Report" button on the Costs tab to print the filtered table

  3. The full P&L Report (Report #1 below) needs to JOIN:
       stock_logs.total_cost  (production costs incurred in period)
     WITH:
       sale_items.unit_price × quantity (actual revenue earned in period)
     TO PRODUCE:
       Actual Realized Gross Profit = Revenue Earned − Production Costs Used

  4. The stock_logs already store profit_per_unit and profit_margin at time
     of distribution (using the price at that moment). A report should use
     THOSE stored values for historical accuracy, not recalculate dynamically.

── SUMMARY: COST RECORDS PAGE vs FULL P&L REPORT ────────────────────────────

  Cost Records Tab (Inventory → Costs):
    Shows: Production cost per distribution batch + potential unit profit
    Good for: Pricing decisions, batch cost review, break-even analysis
    Missing: Time-period filtering, actual sale revenue, printable format

  Full P&L Report (to be built on Reports page):
    Shows: Revenue earned − costs incurred = net profit for a period
    Good for: "Did we make money this month?" business question
    Requires: New Reports page + ReportController + 6 API endpoints


================================================================================
1. WHY ADD PRINTED REPORTS
================================================================================

The KJP Ricemill system already tracks ALL financial data needed for
business reporting. The procurement costs, drying costs, processing yields,
selling prices, discounts, and payment statuses are all stored in the database.

What is MISSING is a way to:
  - View a summarized profit/loss over a period (daily, monthly, yearly)
  - Print or export financial reports for business records or accounting
  - Know at a glance whether the business is profitable or operating at a loss

Adding printed reports does NOT require new data collection — the data
is already there. Reports are simply organized views of existing data.

3. REPORT #1 — PROFIT & LOSS (P&L) STATEMENT  ← MOST IMPORTANT

PURPOSE:
  To answer the core business question: "Did KJP Ricemill make money
  during this period, and how much?"

DATE FILTER:
  - Daily / Weekly / Monthly / Quarterly / Annual
  - Custom date range (from/to picker)

  │              KJP RICEMILL                                           │
  │         PROFIT & LOSS STATEMENT                                     │
  │         Period: January 1, 2026 – January 31, 2026                  │
  ├─────────────────────────────────────────────────────────────────────┤

  │   Gross Sales Revenue                         ₱ XXX,XXX.XX          │

  │   NET REVENUE                                 ₱ XXX,XXX.XX          │

  │   TOTAL COGS                                 (₱  XX,XXX.XX)        │

5. REPORT #3 — PROCUREMENT & RAW MATERIAL COST REPORT
================================================================================

PURPOSE:
  Shows how much was spent on raw palay (unhusked rice) from suppliers
  in a given period. Useful for tracking supply chain input costs.

DATE FILTER: Monthly, Quarterly, Annual, Custom Range

CONTENT:
  - Total procurement batches in period
  - Total kilograms purchased
  - Total sacks purchased
  - Total amount spent (COGS input)
  - Breakdown by supplier:
      Supplier Name   | Qty (kg) | Sacks | Price/kg | Total Cost | Date
  - Breakdown by variety:
      Variety Name    | Qty (kg) | Total Cost | Avg Price/kg
  - Average price per kg (to track inflation/supplier pricing trends)
  - Procurement status summary:
      Pending / Drying / Dried / Completed / Cancelled

DATA SOURCES:
  - procurements table (supplier_id, quantity_kg, sacks, price_per_kg,
    total_cost, variety_id, status, created_at)
  - suppliers table (name)
  - varieties table (name)
  - procurement_batches table (batch_number)

WHERE TO PRINT: Reports page + Print button on Procurement page

6. REPORT #4 — DRYING COST REPORT
================================================================================

PURPOSE:
  Shows the total cost incurred from the drying stage across all batches
  in a period. Drying is a direct operational cost that eats into profit.

DATE FILTER: Monthly, Quarterly, Custom Range

CONTENT:
  - Total drying batches completed
  - Total sacks dried
  - Total days of drying
  - Total drying cost (sum of all drying expenses)
  - Average cost per sack
  - Breakdown per drying record:
      Batch No. | Sacks | Days | Rate/Sack | Total Cost | Status | Date
  - Input vs Output quantity comparison (moisture loss tracking):
      Input kg  vs  Output kg  →  Shrinkage %

DATA SOURCES:
  - drying_processes table (quantity_kg, sacks, quantity_out, days,
    price, total_price, status, started_at, dried_at)
  - procurement_batches table (batch_number)
  - procurements table (linked via pivot)

  FORMULA:
    total_price = (sacks × price_per_sack) × days
    Shrinkage % = ((quantity_kg - quantity_out) / quantity_kg) × 100

WHERE TO PRINT: Reports page + Print button on Drying Process page

7. REPORT #5 — PROCESSING YIELD & EFFICIENCY REPORT
================================================================================

PURPOSE:
  Shows how efficiently the milling stage converts dried palay into
  finished rice. Yield % directly impacts profitability — a lower yield
  means more raw material is wasted.

DATE FILTER: Monthly, Quarterly, Custom Range

CONTENT:
  - Total processing records completed
  - Total input kg (dried rice going in)
  - Total output kg (finished rice coming out)
  - Total husk/waste kg
  - Overall yield percentage
  - Breakdown per processing record:
      Processing ID | Input kg | Output kg | Husk kg | Yield % | Operator | Date
  - Stock distribution after processing (which products received stock)
  - Cost carried through per kg of output rice

DATA SOURCES:
  - processings table (input_kg, output_kg, husk_kg, yield_percent,
    operator_id, status, completed_at)
  - processing_drying_sources table (pivot: processing_id, drying_id, qty)
  - processing_outputs table (product_id, quantity_kg distributed)
  - stock_logs table (tracing stock additions)

  FORMULA:
    yield_percent = (output_kg / input_kg) × 100
    husk_percent  = (husk_kg / input_kg) × 100

WHERE TO PRINT: Reports page + Print button on Processing page

8. REPORT #6 — INVENTORY VALUATION REPORT
================================================================================

PURPOSE:
  Shows the current value of all rice products in stock — i.e., how much
  money is "sitting on the shelves" waiting to be sold.

DATE: As of print date (current snapshot, no date range filter needed)

CONTENT:
  - Per product:
      Product Name | Variety | Current Stock (bags) | Unit Cost | Unit Price
      Stock Value (at cost) | Potential Revenue | Potential Profit
  - Grand totals:
      Total inventory value at cost       : ₱ XX,XXX.XX
      Total potential revenue (at selling): ₱ XX,XXX.XX
      Total potential gross profit        : ₱ XX,XXX.XX
  - Low stock alerts (items below stock_floor threshold)
  - Out of stock items list

DATA SOURCES:
  - products table (name, price, stocks, stock_floor, variety_id)
  - varieties table (name)
  - product cost analysis (computed from processing → drying → procurement
    chain via ProductService::costAnalysis())

  FORMULA:
    Stock Value at Cost    = stocks × unit_cost_per_bag
    Potential Revenue      = stocks × selling_price
    Potential Gross Profit = Potential Revenue - Stock Value at Cost

WHERE TO PRINT: Reports page + Print button on Products page

10. HOW PROFIT IS CALCULATED (FORMULA BREAKDOWN)
================================================================================

The system uses a full supply chain cost trace. Here is the complete path
from raw material purchase to final profit per sale:

  ┌──────────────────────────────────────────────────────────────────────┐
  │  STEP 1: PROCUREMENT COST (Raw Material)                             │
  │    procurement_cost = quantity_kg × price_per_kg                     │
  │    Example: 5,000 kg × ₱10/kg = ₱50,000                             │
  │                                                                      │
  │  STEP 2: DRYING COST                                                 │
  │    drying_cost = (sacks × price_per_sack) × days                    │
  │    Example: (100 sacks × ₱5) × 3 days = ₱1,500                      │
  │                                                                      │
  │  STEP 3: PROCESSING — COST TRACED (No separate fee)                  │
  │    fraction = qty_taken / total_dried_kg                             │
  │    cost_share = fraction × (procurement_cost + drying_cost)          │
  │    unit_cost = cost_share / output_kg                                │
  │                                                                      │
  │  STEP 4: SELLING PRICE (Revenue per unit)                            │
  │    unit_revenue = selling_price_per_bag                              │
  │                                                                      │
  │  STEP 5: PROFIT PER UNIT                                             │
  │    unit_profit = unit_revenue - unit_cost                            │
  │                                                                      │
  │  STEP 6: TOTAL PROFIT (P&L)                                          │
  │    total_revenue = SUM(sales.total) for completed orders             │
  │    total_cogs    = SUM(procurement.total_cost)                       │
  │                  + SUM(drying.total_price)                           │
  │    gross_profit  = total_revenue - total_cogs                        │
  │    gross_margin  = (gross_profit / total_revenue) × 100              │
  └──────────────────────────────────────────────────────────────────────┘

ALREADY STORED IN DATABASE:
  - sales.profit_margin     → Profit margin % per transaction
  - sale_items.total_profit → Profit amount per line item
  - sales.total_cost        → Total cost traced for that sale

The P&L Report reads and aggregates these already-computed values.

QUICK REFERENCE — IS THE BUSINESS PROFITABLE?
================================================================================

  To know if KJP Ricemill made a profit in a given month:

    1. Open Reports page → Profit & Loss tab
    2. Set period to the desired month
    3. Look at GROSS PROFIT line
       - If POSITIVE (green)  → Business made money that month ✓
       - If NEGATIVE (red)    → Business operated at a loss that month ✗
    4. Look at GROSS MARGIN %
       - Industry target for rice milling: 15%–30% margin
       - Below 10% → costs are eating into profit, investigate procurement prices
       - Above 30% → very healthy, business is performing well

  The data for this calculation already exists in the database.
  No new data collection is needed — only a Reports page to display it.
